# SENS

**SENS** — служебное сообщение с **актуальным состоянием окружения** Lira: локальное время, дата, опционально температура GPU/CPU. Модель получает его как отдельную роль в chat template, а не как реплику пользователя.

## Зачем

- Ответы на «который час», «какая дата», «как греется видеокарта» без `web_search`.
- Persona явно помечает sens как **истину с машины** — не перепроверять через интернет.

## Как формируется

Код: `infrastructure/templates/sens_snapshot.py`.

1. **Базовая строка** — `build_sens_status_line()`:
   - время `HH:MM`, дата с названием месяца и дня недели;
   - статус «Активна»;
   - суффикс железа из `nvidia-smi` и `/sys/class/thermal/thermal_zone0` (`sens_hardware_suffix()`).

2. **Дополнение** — `perception_suffix` в `ModelWorker`:
   - краткое «сколько прошло с последней активности» (perception daemon);
   - бит из `WorldState` (например последнее Telegram-событие) — см. [external-events.md](external-events.md).

Сборка в `ChatController._sens_append_for_worker()` → передаётся в worker → вставка в список messages.

## Куда попадает в промпт

В `ModelWorker` при наличии **кастомного** `template_path` (jinja):

- создаётся сообщение `{"role": "sens", "content": …}`;
- вставляется **сразу после** `system` (или первым, если system нет).

Шаблон jinja должен **обрабатывать роль `sens`** (см. `data/models/.../chat_template.jinja`). Без jinja sens не добавляется автоматически.

Подробнее о шаблонах: [chat-templates.md](chat-templates.md).

## Логирование

В `logs/lira.log` / stderr: строки `[SENS]` с длиной и полным текстом (для отладки).

## Связь с инструментами

| Данные | Источник |
|--------|----------|
| Время, дата, GPU/CPU | sens |
| Погода, курсы, новости | `web_search` (если пользователь явно просит) |
| Картинки на диске | `gallery_search` |

Нарушение (модель зовёт web на вопрос про час) правится **persona**, не отдельным кодом sens.

## Настройка

Отдельного пункта в `config.json` для sens нет: время всегда локальное ОС; GPU — если доступен `nvidia-smi`.

Расширение (новые датчики) — правка `sens_snapshot.py` и jinja-шаблона под новый формат строки.
