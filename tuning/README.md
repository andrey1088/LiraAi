# Tuning: датасет Лиры (DPO / preference)

Каталог для дообучения поведения «не поддакивать»: пары **chosen** (твёрдый ответ Лиры) / **rejected** (поддакивание).

## Файлы

| Файл | Назначение |
|------|------------|
| `lira-dataset.csv` | Основной датасет для обучения (`instruction`, `input`, `chosen (Твердый ответ)`, `rejected (Поддакивание)`) |
| `dpo_dataset.csv` | Исходник с Hugging Face (англ.): `user_input`, `chosen`, `rejected` |
| `merge_chunks.py` | Склеить `generated/chunk_*.json` с существующим `lira-dataset.csv` |
| `expand_lira_dataset.py` | Автодобавление строк: перевод + стилизация `chosen` (без агентов) |
| `lira_style.py` | Эвристическая стилизация переведённого `chosen` под голос Лиры |
| `fix_english_inputs.py` | Дочистка: перевести `input`/`rejected`, если остался английский |
| `generated/chunk_*.json` | Промежуточные чанки (не коммитятся, см. `.gitignore`) |

**Системный промпт** (`instruction`) — один на все строки, как в первой записи `lira-dataset.csv`:

```text
Ты — Лира, персональный ИИ-ассистент Андрея. Думай и говори о себе в женском роде, ...
```

## Текущий объём

- **70** строк — ручные (русские провокации + стиль Лиры).
- **+230** — из первых 230 строк `dpo_dataset.csv` (перевод + переписанный `chosen`).
- **Итого: 300** строк в `lira-dataset.csv`.

Перед обучением имеет смысл выборочно вычитать новые строки и добавить свои примеры под реальные диалоги.

## Воркфлоу A: дополнить датасет из английского CSV (рекомендуется для качества)

1. Положить или обновить `dpo_dataset.csv` (`user_input`, `chosen`, `rejected`).
2. Убедиться, что в `lira-dataset.csv` есть эталонные строки с нужным `instruction`.
3. Сгенерировать чанки (вручную, через агента или редактор) в `generated/chunk_N.json` — массив объектов с ключами:
   - `instruction` — копия из `lira-dataset.csv`
   - `input` — вопрос/утверждение на русском в «…»
   - `chosen (Твердый ответ)` — жёсткий ответ Лиры в «…»
   - `rejected (Поддакивание)` — поддакивание на русском в «…»
4. Склеить:

```bash
cd ~/Lira2/tuning
python3 merge_chunks.py
```

5. При необходимости исправить английские поля:

```bash
python3 fix_english_inputs.py
```

6. Выборочная вычитка → обучение → тест 20–30 провокаций.

`merge_chunks.py` **добавляет** все `generated/chunk_*.json` к строкам, уже лежащим в `lira-dataset.csv`. Чтобы начать с нуля, сохраните копию или временно оставьте в CSV только заголовок и эталонные строки.

## Воркфлоу B: автоматически (перевод + эвристика)

Без агентов; `chosen` слабее, чем в ручной правке.

```bash
cd ~/Lira2/tuning
# из корня репозитория, с venv:
~/Lira2/venv/bin/python3 expand_lira_dataset.py 230
```

Опции:

- Число аргументом — сколько **первых** строк брать из `dpo_dataset.csv`.
- Кэш перевода: `.expand_cache.json` (в `.gitignore`).
- Ollama (опционально, медленно; нужна локальная модель, например `ava:latest`):

```bash
~/Lira2/venv/bin/python3 expand_lira_dataset.py 230 --ollama
```

По умолчанию Ollama **не** вызывается — только перевод и `lira_style.py`.

Финальная сборка 300 строк в репозитории сделана через **воркфлоу A** (чанки + `merge_chunks.py`), не через Ollama.

Зависимости: `pip install deep-translator` (уже в venv проекта).

## Воркфлоу C: только дочистка английского

После любой сборки, если в `input` или `rejected` остался английский:

```bash
python3 fix_english_inputs.py
```

Скрипт сопоставляет строки с индексом **≥ 71** с соответствующими строками `dpo_dataset.csv` (нулевая строка DPO = 71-я строка датасета после 70 исходных).

## Заметки по обучению

- **300 строк** — разумный первый эксперимент, для устойчивого эффекта на 26B часто нужно больше разнообразия и/или правки промпта в чате.
- Добавляйте свои русские примеры под темы, которых нет в DPO (работа, код, отношения, быт).
- Дубли в DPO (один `user_input`, несколько paraphrase) в датасете дают несколько вариантов `chosen` на одну тему — для DPO это допустимо.
